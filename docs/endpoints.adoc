== Dataframe References

The API uses immutable dataframe references that typically look like:

----
nsoud-search-session:<session_id>:decisions
nsoud-search-session:<session_id>:hits
nsoud-search-session:<session_id>:arguments
----

Expected kinds by endpoint family:

- `decisions/*` typed dataframe endpoints expect a `:decisions` ref
- `hits/*` typed dataframe endpoints expect a `:hits` ref
- `arguments/*` typed dataframe endpoints expect an `:arguments` ref
- generic `/dataframe/describe` accepts any dataframe ref

== Search Result Limits

Hybrid search endpoints expose two separate knobs:

- `limit` controls how many ranked results are materialized into the search session and dataframe refs
- `top_rows_limit` controls how many preview rows are returned inline in the immediate response

This separation applies to:

- `POST /search`
- `POST /decisions/search-within-dataframe`

== Typed Request Contracts

The public contract is intentionally strict:

- decision, hit, and argument dataframe endpoints use endpoint-specific request DTOs
- field names, enum values, and allowed filters differ by endpoint family
- decision semantic and fulltext search take `identifier` from the path, not from the JSON body

Use the generated OpenAPI spec in `openapi/fragmea-api.openapi.yaml` as the source of truth for field-level request and response details.
