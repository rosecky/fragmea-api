== Dataframe References

The API uses immutable dataframe references that typically look like:

----
nsoud-search-session:<session_id>:decisions
nsoud-search-session:<session_id>:hits
nsoud-search-session:<session_id>:arguments
----

Expected kinds by endpoint family:

- `decisions/*` typed dataframe endpoints expect a `:decisions` ref
- `hits/*` typed dataframe endpoints expect a `:hits` ref
- `arguments/*` typed dataframe endpoints expect an `:arguments` ref
- generic `/dataframe/*` endpoints accept any dataframe ref

== Filtering And Aggregation Conventions

Generic operations use the common shapes:

- filter item: `{ "field": "...", "op": "eq|ne|in|contains|gte|lte|gt|lt", "value": ... }`
- sort item: `{ "field": "...", "direction": "asc|desc" }`
- group-by item: `{ "field": "...", "label": "...", "date_part": "year|month|day" }`
- metric item: `{ "op": "count|sum|avg|min|max|percentage|percent|share|list|list_distinct", "field": "...", "alias": "..." }`

Typed decision, hit, and argument endpoints restrict field names and enum values more tightly. Use the OpenAPI spec in `openapi/nsoud-search-stack-api.openapi.yaml` for field-level details.
